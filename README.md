# pXcontrol

Управление Telegram-каналами с десктопа: отложенный постинг, видео-конвейер
на ffmpeg, генерация контента ИИ, парсинг источников.

> 🟢 **Ранний прототип.** Аккаунты (боты, userbot, ключи ИИ) и подключение
> каналов уже работают; постинг и видео-конвейер — в разработке.

## Зачем

Главная боль при ведении видео-каналов — ручная подготовка каждого ролика
(формат, вотермарк, превью). `pXcontrol` строится вокруг видео-центричного
конвейера: загрузил исходник → получил готовый пост в канале по расписанию.

## Что уже работает

- Десктоп-приложение (PySide6 + QFluentWidgets): тёмная/светлая тема,
  боковая навигация.
- Движок в фоновом потоке — окно не блокируется на сетевых операциях.
- SQLite с миграциями Alembic; **секреты шифруются** (Fernet), ключ —
  в системном хранилище ОС (keyring).
- Боты: добавление с живой проверкой токена (`getMe`), несколько ботов,
  диагностика «где состоит бот» по событиям Telegram.
- Userbot (MTProto): пошаговый вход — код из Telegram, пароль 2FA;
  сессия хранится зашифрованной. Отдельный аккаунт, не основной.
- Каналы: подключение с проверкой прав (канал существует, бот — админ
  с правом публиковать), понятные ошибки.
- Посты: «сейчас» — через бота; отложенные создаются **прямо в канале**
  (серверное планирование Telegram) — публикуются даже при выключенном
  приложении; источник истины — сам канал, страница «Расписание» читает
  отложенные из Telegram.
- Файл логов с ротацией (`logs/pxcontrol.log`).
- **Подготовка видео** (порт `makeVideo`): FullHD, вотермарк, заставка
  для превью, обложка; пресеты обработки; страница «Видео». Результаты —
  в `media/processed/`.

## Дорожная карта (кратко)

1. Публикация видео: подготовленный файл → «сейчас» через бота или
   отложенно через userbot; пресет по умолчанию для канала.
2. Источники (RSS, каналы через MTProto) и очередь модерации.
3. Генерация текстов ИИ (Claude API).

Полная карта — [docs/09-roadmap/roadmap.md](docs/09-roadmap/roadmap.md).

## Технологии

Python 3.11+ (строгая типизация, mypy strict) · PySide6 + QFluentWidgets ·
aiogram 3 (Bot API) · Telethon (MTProto) · SQLAlchemy 2 async + Alembic ·
cryptography + keyring · pytest · ruff.

Полный стек с обоснованием каждого выбора —
[docs/02-architecture/tech-stack.md](docs/02-architecture/tech-stack.md).

## Структура кода

```
src/pxcontrol/
  __main__.py          точка входа (python -m pxcontrol)
  app.py               сборка и порядок запуска движка и интерфейса
  config/              бутстрап-настройки (.env); остальной конфиг — в БД
  engine/              движок: UI-независимая бизнес-логика
    engine.py          оркестрация компонентов
    worker.py          фоновый поток + цикл asyncio (мост к интерфейсу)
    db/                SQLAlchemy 2.0 (async), SQLite, миграции Alembic
    security/          шифрование секретов (Fernet + keyring)
    telegram/          шлюз: Bot API (aiogram) + MTProto (Telethon)
    video/             подготовка видео — чистый модуль поверх ffmpeg
    services/          сервисы бизнес-логики (аккаунты, каналы, посты, видео)
  ui/                  интерфейс на PySide6 (страницы, мост к движку)
tests/                 тесты (pytest, без походов в сеть)
docs/                  документация проекта (точка входа — CLAUDE.md)
```

Порядок запуска и модель потоков —
[docs/02-architecture/runtime.md](docs/02-architecture/runtime.md).

## Запуск

```bash
# окружение и зависимости (менеджер — uv; подойдёт и pip + venv)
uv venv && uv pip install -e ".[dev]"

# настройки: скопировать шаблон (секреты в .env не хранятся — они в БД)
cp .env.example .env

# запуск приложения
python -m pxcontrol

# тесты, типы, линт
pytest -q && mypy src/pxcontrol && ruff check src tests
```

Требуется работающее системное хранилище ключей (GNOME Keyring/KWallet) —
в нём живёт ключ шифрования секретов — и `ffmpeg`/`ffprobe` в системе
(подготовка видео; путь настраивается через `FFMPEG_PATH`).
