Формат — [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/),

## [Unreleased]

### Добавлено

- Хранение настроек (ADR-0013): таблицы «строка = имя → значение»
  на владельца — `app_settings` (приложение) и `channel_settings`
  (канал, внешний ключ с каскадом); состав, типы и умолчания задаёт
  типизированный реестр ключей, битые значения откатываются
  к умолчанию. Первые настройки: тема оформления и состояние окна
  переживают перезапуск, «Публикация» предвыбирает канал прошлой
  публикации, путь к ffmpeg правится на странице «Настройки»
  (пусто — бутстрап из `.env`; смена — без перезапуска, сервисы
  получают путь провайдером). Зарезервированная таблица `settings`
  переименована в `app_settings` и наконец используется; колонка-заделка
  `channels.video_preset_id` удалена — её роль займёт настройка канала
  `default_video_preset` (миграция `a5d8f31c9b27`).
- Затухание на краях ролика: «Затухание в начале, с» и «Затухание
  в конце, с» в разделе «Вывод» (0 — без эффекта; хранится в пресете,
  миграция `b9e2c74a51d3`). Появление из чёрного / уход в чёрное,
  видео и звук вместе; со звуком при заставке эффект стартует после
  тишины задержки. Самостоятельный эффект — к обрезке не привязан,
  но отлично маскирует срез.
- Обрезка видео с краёв: в параметрах обработки — «Отрезать в начале, с»
  и «Отрезать в конце, с» (доли секунды; 0 — не резать; хранится
  в пресете, миграция `d2b8e51c7f94`). Срез точный (перекодирование),
  все остальные параметры считаются от обрезанной версии: окно
  вотермарка, момент кадра заставки, случайные кадры и кандидаты —
  из обрезанного диапазона; обрезка больше длительности — понятная
  ошибка до кодирования.
- Очередь отправки (ADR-0012): «Отправить» ставит пост в очередь
  и сразу освобождает форму — пока большое видео уходит в Telegram,
  следующие посты готовятся и отправляются без ожидания. Посты уходят
  последовательно; очередь видна на «Публикации» (карточка на элемент:
  статус, полоса прогресса у отправляющегося, «Отмена» у каждого —
  включая обрыв активной загрузки). Успех исчезает с плашкой, ошибка
  остаётся до «Убрать»; ошибка одного поста не трогает остальные.
  Очередь в памяти (истина — канал, ADR-0010): при закрытии окна
  с неотправленным — подтверждение выхода.
- Редактор словаря поля подписи: в «Полях подписи…» у каждого поля —
  кнопка «Словарь…» (список значений с удалением — опечатки
  и устаревшее больше не живут вечно; добавление пачкой через запятую,
  дубли без учёта регистра отбрасываются, как при автопополнении).
- Кнопка просмотра исходника на странице «Видео»: рядом с «Обзор…»,
  открывает выбранный файл системным плеером (не искать в проводнике);
  активна, только когда путь указывает на существующий файл.
- Подключение канала через userbot: аккаунт — админ канала, бот
  не нужен (выбор способа в диалоге подключения). Бот-путь остался;
  при нём попутно проверяется userbot, и карточка канала показывает
  способы администрирования: «бот», «userbot» или оба.
- Публикация по возможностям канала: userbot в приоритете (всё,
  до 2 ГБ, отложенные); канал «только бот» публикуется через Bot API —
  текст и медиа до 50 МБ, только «сейчас» (расконсервированы
  `send_text`/`send_media`); без способов — понятная ошибка. Форма
  «Публикации» адаптируется: строка возможностей под каналом,
  отложка блокируется с подсказкой.
- Управление доступами подключённого канала: «Проверить доступы»
  (актуализирует флаг userbot в обе стороны, сообщает о потере прав
  бота), «Назначить бота…» каналу без бота (с проверкой прав)
  и «Отвязать бота».
- Комментарий в метаданные видео: поле «Комментарий (метаданные)»
  в параметрах обработки — свободный текст (обычно «ссылка на канал —
  описание») пишется тегом `comment` контейнера MP4 при кодировании;
  виден в свойствах файла и плеерах, переживает пересылку и перезалив.
- Плавное появление/исчезание вотермарка («плавность, с» в строке
  окна показа; 0 — резко): fade по альфа-каналу на краях окна
  с ненулевыми отступами, дружит с заставкой; переходы, не влезающие
  в окно, — ошибка до кодирования.
- Окно показа вотермарка в диалоге пресета: появление через N секунд
  от начала и скрытие за N секунд **до конца** ролика (0 — без
  ограничения). Отступы переносимы между роликами разной длины;
  не помещающиеся в ролик — ошибка до кодирования (поля переименованы
  в `wm_start_offset`/`wm_end_offset`, миграция `f2b8d61c3a97`).
- Заставка «случайные кадры на выбор»: при обработке открывается плитка
  из N случайных кадров видео (по умолчанию 6, «Обновить» — новая
  партия), выбранный кадр уходит в заставку как есть — кандидаты
  извлекаются один раз в финальном качестве, без повторного
  выдёргивания по времени.
- Переименование файла при отправке: у шаблона подписи — необязательный
  шаблон имени файла (`{Author}, {title} ({Genre}) {quality}
  (@{channel})`); после сборки подписи имя вычисляется движком
  (значения полей без решёток, качество из ffprobe, @канала, очистка
  символов) и правится в строке «Переименовать при отправке»;
  при публикации файл и его кадр-превью переименовываются на диске.
- Шаблоны подписей к постам: у канала — пул полей («Genre», «Year»…)
  со словарями значений (общими для всех шаблонов) и именованные
  шаблоны из этих полей. На «Публикации» — «Собрать подпись…»
  (жирное название из имени файла + строки «Поле: #значения», ненужные
  поля отключаются, словари пополняются сами) и «Поля подписи…»
  (настройка полей и шаблонов, порядок — перетаскиванием).
- Нормальное превью у публикуемого видео: конвейер сохраняет кадр-превью
  рядом с результатом (`<имя>.png` — кадр заставки/обложки или первый
  кадр), при отправке из него режется JPEG-миниатюра, вписанная
  в 320×320 с сохранением пропорций; для видео без превью-соседа —
  случайный кадр из середины. Метаданные видео (длительность, размеры)
  Telethon извлекает через новую зависимость `hachoir` — без них
  Telegram игнорирует миниатюру и рисует плитку с неверными пропорциями.
- Страница «Публикация» — единая точка создания постов (ADR-0011):
  тип контента сегментами (текст, фото, видео, аудио, файл), канал,
  текст/подпись, «сейчас» или дата+время, прогресс загрузки файла.
  Весь контент публикует userbot (лимит Bot API — 50 МБ на файл,
  у MTProto — 2 ГБ); в движке — единая сущность `PostDraft`
  и `PostsService.publish()`; путь через бота законсервирован для ИИ.
  Кнопка «Опубликовать…» на карточке результата «Видео» открывает
  «Публикацию» с предзаполненным файлом; «Расписание» — только список
  отложенных.
- Выбор качества видео в пресете: целевой битрейт в Мбит/с или
  «как в оригинале» (по умолчанию — битрейт исходника; если ffprobe
  его не сообщил — постоянное качество CRF 20).
- Прогресс обработки видео с процентами: ffmpeg отчитывается через
  `-progress pipe:1`, полоса и подпись «Кодирование: NN%» на странице
  «Видео».
- Подготовка видео (порт из `makeVideo`, слайс A): чистый модуль
  `engine/video/` (FullHD, вотермарк с окном показа, заставка для превью,
  обложка), пресеты обработки (CRUD), страница «Видео» с запуском
  обработки и результатами в `media/processed/`. Публикация видео —
  следующий слайс.
- Первый пост: публикация «сейчас» через бота; отложенные посты
  создаются прямо в канале (серверное планирование Telegram через
  userbot, ADR-0010); страница «Расписание» читает отложенные из
  Telegram. Userbot подключается при старте и сразу после входа.
- Логирование в файл с ротацией: `logs/pxcontrol.log` (~2 МБ × 5),
  уровень через `LOG_LEVEL`; ошибки операций движка пишутся с полным
  трейсбеком; в лог попадает контекст подключения канала (бот, ввод,
  нормализация).
- Диагностика «Где состоит бот?» на карточке бота: события Telegram
  за 24 часа (добавления в каналы/группы и права) — в диалоге и в логе.

### Изменено

- «Настройки» — категориями: слева список («Общие», «Аккаунты»),
  справа правка выбранной категории. «Аккаунты» больше не всплывающий
  дочерний пункт боковой навигации окна — встроены категорией.

- Рефакторинг (16.07.2026): один тип `ChannelInfo` на границе
  телеграм-слоя вместо двух одинаковых (`ChannelInfo` бота
  и `UserbotChannelInfo`); общий помощник превью текста `text_preview`
  вместо двух копий обрезки «с троеточием»; каркас прокручиваемой
  страницы — один помощник `page_layout` (отступы `PAGE_MARGINS`)
  вместо семи одинаковых строк на пяти страницах; из dev-зависимостей
  удалён неиспользуемый `respx` (мокает httpx, а транспорты —
  aiogram/Telethon); документация сверена с кодом: `data-schemas.md`
  переписан под фактические датаклассы границы (план с Pydantic
  отменён), статусы готовых доков подняты до 🟢, поправлены реликты
  в `CLAUDE.md` и ссылка на внешний `makeVideo/SPEC.md`.
- Затухание — отдельной карточкой под «Обрезкой» (было строкой
  в «Выводе»): края включаются чекбоксами «В начале»/«В конце»,
  длительность — в полях рядом (по умолчанию 2 с; поле активно при
  включённом чекбоксе, выключенный край хранится в пресете нулём).
- Рефакторинг (15.07.2026): удалён законсервированный `send_now`
  (его сценарий полностью покрыл запасной бот-путь `publish`);
  `MediaKind` и новый тип `ScheduledMessage` — на границе телеграм-слоя
  (сырые сообщения Telethon не доезжают до сервисов); перевод ошибок
  aiogram — одним контекст-менеджером вместо пяти копий; ошибка пароля
  2FA закрывает незавершённый вход, как и ошибка кода; умолчания пресета
  видео — в одном месте (`PresetFields`) вместо трёх; параметры
  кодирования и запуск ffmpeg/ffprobe — общие (`constants.py`,
  `engine/video/ffmpeg.py`); миниатюра «для Telegram» переехала из
  чистого видео-модуля в слой публикации; включён набор правил ruff
  (I, N, UP, B, BLE, SIM…) — исключения переименованы по конвенции
  (`…Error`); в интерфейсе — общие помощники ошибок и выбора файла,
  лимиты размеров файлов берутся из констант движка; документация
  сверена с кодом (12 расхождений устранено).
- Страница «Видео»: параметры обработки — прямо на странице,
  «Обработать» применяет то, что на экране, без сохранения; пресет —
  «загрузчик» (выбор заполняет панель, «Сохранить»/«Сохранить как…»/
  «Удалить» — явные действия) и теперь необязателен. Диалог пресета
  и список карточек удалены — меньше телодвижений на потоке видео
  с мелкими правками под каждый ролик.
- Рефакторинг (14.07.2026): единая сущность исходящего поста
  `OutgoingPost` на границе «сервисы → транспорты» (вместо семи
  параметров в трёх сигнатурах); общая панель прогресса `ProgressPanel`
  вместо двух копий моста и полосы; один способ поиска ffprobe
  (`ffprobe_bin_for`) вместо двух дублей; единый стиль показа ошибок
  на страницах и в диалогах; документация приведена к коду
  (модель данных, шлюз, расписание, компоненты, стек).
- Рефакторинг (12.07.2026): мост «интерфейс → движок» принимает колбэки
  без фиктивных параметров; единый строитель карточек-строк и показ
  ошибок на всех страницах.

### Удалено

- Заглушки разделов «Очередь», «Источники», «Генерация» из навигации —
  вернутся вместе с реальными функциями.
- Мёртвый код: пакет планировщика (APScheduler вернётся с парсингом
  источников), класс `BotApiTransport`, заглушки `read_channel`
  и `VideoProcessor`, настройка `TZ`, неиспользуемые зависимости
  (`httpx`, `feedparser`, `anthropic` — вернутся со своими функциями).
  `FFMPEG_PATH` тоже удалялась, но вернулась вместе с подготовкой
  видео — сейчас это действующая настройка.

### Исправлено

- Имя файла по шаблону обрезалось на произвольных 150 символах —
  длинные названия теряли хвост ещё при формировании. Теперь предел —
  в байтах UTF-8 (240, с учётом расширения), как того требуют файловые
  системы (ext4 — 255 байт на имя): латиница получила ~236 символов,
  кириллица режется по честному байтовому пределу без битых символов
  (раньше 150 кириллических букв = 300 байт — файл не переименовался бы
  вовсе).
- Обложка уводила индекс `moov` в конец файла: главный проход ставил
  `-movflags +faststart` (moov в начало — для перезалива и проигрывания
  «на лету»), но шаг вшивания обложки ремуксил без флага и возвращал
  moov в хвост. Теперь `+faststart` есть и на вшивании обложки.
- Цвета вотермарка искажались (красный зеленел): overlay по умолчанию
  смешивает с цветностью в четверть разрешения (yuv420), и тонкие
  штрихи перенимали цвет соседей, а RGB→YUV шёл матрицей bt601 вместо
  bt709. Теперь конверсия — явно bt709, смешивание — в yuv444.
- Заставка из своей картинки валила обработку, если пропорции картинки
  отличались от видео: `xfade` требует одинаковых размеров входов.
  Теперь размер кадра считается в Python (`fitted_size`) и задаётся
  явно и видео, и заставке; чужая картинка вписывается с чёрными
  полями по центру. В диалоге пресета поля заставки активны только
  при включённом переключателе — раньше выключенная заставка выглядела
  настроенной.
- Вотермарк накладывался с искажением пропорций и размера: устаревший
  фильтр `scale2ref` в ffmpeg 8 считает размеры неверно — заменён
  на `scale` со вторым входом-эталоном (`rw`, высота `-2` сохраняет
  пропорции). Заодно стала наглядно работать прозрачность вотермарка.
- Подключение канала: ID с пробелами внутри и ссылки `t.me/c/…`
  распознаются; инвайт-ссылки (`t.me/+…`) отклоняются с пояснением;
  в ошибках теперь виден оригинальный ответ Telegram; подсказка
  в диалоге — как узнать ID приватного канала.

## [0.1.0] — 2026-07-12

Первая версия: рабочий прототип.

### Добавлено

- Каркас приложения: движок (фоновый поток + asyncio) и интерфейс
  (PySide6 + QFluentWidgets, тёмная/светлая тема, тиловый акцент).
- SQLite с миграциями Alembic; БД и `.env` — в папке приложения
  (портативный режим).
- Шифрование секретов: Fernet, ключ — в системном хранилище ОС (keyring).
- Страница «Настройки → Аккаунты»: боты (живая проверка токена через
  `getMe`), userbot MTProto (пошаговый вход: код, 2FA; сессия шифруется),
  ключи ИИ.
- Страница «Каналы»: подключение с проверкой прав бота (админ + право
  публиковать), список, удаление.
- Планировщик (APScheduler) с догоном пропущенного при старте — каркас.
- Документация: видение, архитектура (9 ADR), модули, модель данных.
