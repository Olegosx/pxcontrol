# Схемы данных (типы границы движка)

> Статус: 🟢 готов · Обновлено: 2026-07-16 · Владелец: olegosx

## Назначение документа

Описать форматы данных движка — структуры, которыми оперируют его сервисы
и которые показывает интерфейс.

## Как реализовано

Все типы границы — **неизменяемые датаклассы** (`@dataclass(frozen=True)`),
а не Pydantic-модели: данные рождаются внутри приложения (БД, Telegram),
внешнего непроверенного ввода на этой границе нет, поэтому валидация
Pydantic не нужна, а неизменяемость защищает от случайных правок при
передаче между потоками (интерфейс ↔ движок). Pydantic используется
в одном месте — bootstrap-настройки `pxcontrol.config.Settings`
(разбор `.env`, см. [configuration.md](../06-config/configuration.md)).

Соглашение об именах: `*Dto` — снимок для показа в интерфейсе;
остальные имена — входные сущности и внутренние типы границы.

## Типы по слоям

### Сервисы (`engine/services/*`) — то, что видит интерфейс

| Тип | Модуль | Роль |
| --- | --- | --- |
| `BotDto`, `TgAccountDto`, `AiKeyDto` | `accounts.py` | аккаунты: боты, userbot, ключи ИИ |
| `ChannelDto`, `ChannelAccess` | `channels.py` | канал и итог перепроверки доступов |
| `PostDraft` | `posts.py` | черновик публикации — единый вход для всех типов контента |
| `PublishCapabilities`, `ScheduledPostDto` | `posts.py` | возможности канала; отложенная запись из Telegram |
| `QueueItemDto`, `QueueItemStatus` | `publish_queue.py` | элемент очереди отправки и его статус |
| `PresetDto`, `PresetFields`, `FrameCandidate` | `video.py` | пресет обработки видео и кадры-кандидаты |
| `FieldDto`, `TemplateDto`, `TemplateFieldDto`, `CaptionLine` | `captions.py` | поля со словарями и шаблоны подписей |

### Телеграм-слой (`engine/telegram/types.py`) — граница «сервисы → транспорты»

| Тип | Роль |
| --- | --- |
| `MediaKind` | тип вложения поста (текст/фото/видео/аудио/файл) |
| `ChannelInfo` | канал, проверенный любым транспортом (бот или userbot) |
| `OutgoingPost` | исходящий пост для транспорта |
| `ScheduledMessage` | отложенная запись, прочитанная из Telegram |

### Видео-конвейер (`engine/video/*`) — чистый модуль без Telegram

| Тип | Модуль | Роль |
| --- | --- | --- |
| `ProcessingOptions` | `pipeline.py` | полный набор параметров обработки |
| `WatermarkOptions`, `FilterGraph` | `filtergraph.py` | вотермарк и собранный граф фильтров ffmpeg |
| `VideoInfo` | `probe.py` | сведения об исходнике (ffprobe) |

## Правило

Типы границы не равны напрямую таблицам БД: внутренняя модель хранения
описана отдельно в [data-model.md](../05-data/data-model.md). Здесь — то,
что видят интерфейс и транспорты. Сырые объекты внешних библиотек
(Telethon, aiogram, JSON ffprobe) через границу не проходят —
транспорты переводят их в перечисленные типы.
