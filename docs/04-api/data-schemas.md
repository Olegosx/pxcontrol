# Схемы данных (запросы/ответы)

> Статус: 🟡 черновик-заглушка · Обновлено: 2026-06-30 · Владелец: olegosx

## Назначение документа

Описать форматы данных движка — структуры, которыми оперируют его сервисы
и которые показывает интерфейс. Реализуются как модели Pydantic (валидация
и типизация).

## Схемы (заполнить)

- [ ] `ChannelIn` / `ChannelOut` — канал.
- [ ] `PostIn` / `PostOut` — пост и его расписание.
- [ ] `QueueItem` — элемент очереди модерации.
- [ ] `SourceIn` / `SourceOut` — источник контента.
- [ ] `GenerateRequest` / `GenerateResult` — задание и результат генерации ИИ.
- [ ] `ErrorResponse` — единый формат ошибки.

## Правило

Схемы API не равны напрямую таблицам БД: внутренняя модель хранения
описана отдельно в [data-model.md](../05-data/data-model.md). Здесь — то,
что видит клиент.
