# API движка (внутренний интерфейс)

> Статус: 🟢 готов · Обновлено: 2026-07-16 · Владелец: olegosx

## Назначение документа

Описать программный интерфейс **движка** (engine) — набор операций, которые
интерфейс на PySide6 вызывает напрямую в процессе. Это десктопное приложение
([ADR-0006](../02-architecture/decisions/0006-desktop-pyside6.md)), поэтому
HTTP/REST API не используется; «API» здесь — публичные Python-сервисы движка.

> Если в будущем понадобится удалённое управление, этот же движок можно
> обернуть в HTTP API (тогда вернётся вопрос FastAPI). Сейчас — вне рамок.

## Сервисы движка (реализованы)

| Сервис | Назначение |
|---|---|
| `AccountsService` | боты (проверка токена, диагностика), userbot (вход: код/2FA), ключи ИИ |
| `ChannelsService` | подключение каналов (userbot или бот), доступы, список, удаление |
| `PostsService` | публикация `PostDraft` по возможностям канала: userbot в приоритете, бот — запасной; отложки — сервер Telegram; чтение отложенных (ADR-0010, ADR-0011) |
| `VideoService` | пресеты обработки, запуск подготовки видео (чистый модуль `engine/video/`), действующие папки видео (`dirs_for`, `processed_dir_for_channel`) |
| `CaptionsService` | поля со словарями, шаблоны подписей, сборка текста и имени файла |
| `PublishQueue` | очередь отправки: последовательная публикация, прогресс, отмена (ADR-0012) |
| `SettingsService` | настройки приложения и каналов: реестр ключей, кэш, провайдер ffmpeg (ADR-0013) |

Каждый сервис объявляет узкий протокол нужной ему части шлюза Telegram
(разделение интерфейсов) — тесты подменяют только его.

## Появятся позже

`SourcesService` (источники), `QueueService` (модерация),
`GenerationService` (ИИ) — по дорожной карте.
