# API движка (внутренний интерфейс)

> Статус: 🟡 черновик-заглушка · Обновлено: 2026-06-30 · Владелец: olegosx

## Назначение документа

Описать программный интерфейс **движка** (engine) — набор операций, которые
интерфейс на PySide6 вызывает напрямую в процессе. Это десктопное приложение
([ADR-0006](../02-architecture/decisions/0006-desktop-pyside6.md)), поэтому
HTTP/REST API не используется; «API» здесь — публичные Python-сервисы движка.

> Если в будущем понадобится удалённое управление, этот же движок можно
> обернуть в HTTP API (тогда вернётся вопрос FastAPI). Сейчас — вне рамок.

## Сервисы движка (реализованы)

| Сервис | Назначение |
|---|---|
| `AccountsService` | боты (проверка токена, диагностика), userbot (вход: код/2FA), ключи ИИ |
| `ChannelsService` | подключение каналов с проверкой прав бота, список, удаление |
| `PostsService` | «сейчас» через бота; отложки через сервер Telegram; чтение отложенных (ADR-0010) |

Каждый сервис объявляет узкий протокол нужной ему части шлюза Telegram
(разделение интерфейсов) — тесты подменяют только его.

## Появятся позже

`VideoService` (обработка видео), `SourcesService` (источники),
`QueueService` (модерация), `GenerationService` (ИИ) — по дорожной карте.
