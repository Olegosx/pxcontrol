# Модуль «Подготовка видео»

> Статус: 🟢 реализовано (подготовка) · Обновлено: 2026-07-13 · Владелец: olegosx

## Назначение

Готовит видеофайл к публикации в Телеграме: FullHD-вписывание, вотермарк,
заставка для красивого превью, обложка. Порт наработок из референса
`makeVideo` (поведение — по его `SPEC.md`).

## Архитектурная граница (важно)

**Подготовка отделена от публикации.** Контракт — путь к файлу:

- `engine/video/` — **чистый модуль**: файл + `ProcessingOptions` →
  обработанный файл. Не знает про Telegram, каналы и БД. Синхронный
  (как и его тесты); async-граница — в сервисе (`asyncio.to_thread`).
- `services/video.py` — `VideoService`: пресеты в БД (`video_presets`),
  маппинг пресета в `ProcessingOptions`, пути (`media/processed/`
  в папке приложения), человеческие ошибки.
- Публикация готового файла — зона `PostsService.publish` (единая
  сущность `PostDraft`, ADR-0011): кнопка «Опубликовать…» на карточке
  результата открывает страницу «Публикация» с предзаполненным файлом.
- Конвейер сохраняет кадр-превью рядом с результатом (`<имя>.png`:
  кадр заставки/обложки, иначе первый кадр). При публикации из него
  режется JPEG-миниатюра ≤320×320 с сохранением пропорций (квадрат
  нельзя — Telegram растягивает миниатюру до пропорций видео); для
  чужого видео без превью — случайный кадр из середины. Метаданные
  для Telegram извлекает `hachoir` (иначе миниатюра игнорируется).

## Состав чистого модуля

| Файл | Что делает |
|---|---|
| `probe.py` | метаданные через ffprobe (размеры, длительность, fps, звук, битрейт) |
| `filtergraph.py` | сборка `-filter_complex`: FullHD, вотермарк с окном показа, заставка hold+xfade — чистые функции |
| `frames.py` | кадр заставки: random-middle / first / time / frame / image |
| `pipeline.py` | сборка и запуск команды ffmpeg, вшивание обложки (`attached_pic`) |

## Пресеты

Таблица `video_presets` (все параметры makeVideo). Управляются со страницы
«Видео»: создание/правка/удаление; при обработке выбираются из списка.

Качество видео: поле `video_bitrate_kbps` — целевой битрейт; NULL
(по умолчанию) — «как в оригинале»: берётся битрейт исходника из ffprobe,
а если тот его не сообщил — постоянное качество CRF 20.

Вотермарк масштабируется под кадр с сохранением пропорций: `scale`
со вторым входом-эталоном (`rw`, высота `-2`). Устаревший `scale2ref`
не используется — в ffmpeg 8 он искажает размер и пропорции.

Цвета вотермарка: RGB→YUV — явно матрицей bt709 (по умолчанию ffmpeg
берёт bt601, и оттенки сдвигались), смешивание overlay — в `yuv444`
(по умолчанию yuv420: цветность в четверть разрешения, тонкие штрихи
перенимали цвет соседей). Остаточное ограничение — сам формат H.264
yuv420p: у штрихов в 1–2 пикселя цветность частично теряется при любом
кодировании; лекарство — крупнее масштаб вотермарка или жирнее шрифт.

Размер итогового кадра считается в Python (`fitted_size`, вписывание
в FullHD с чётными сторонами): основное видео и кадр заставки приводятся
к одному явному размеру — фильтр склейки `xfade` требует точного
совпадения размеров входов. Своя картинка-заставка с иными пропорциями
вписывается с чёрными полями по центру (letterbox), а не валит обработку.
Привязка «пресет по умолчанию для канала» (`channels.video_preset_id`) —
заиграет в слайсе B.

## Выбор кадра заставки

Режимы источника кадра: случайный из середины, **случайные кадры
на выбор**, момент времени, своя картинка. В режиме «на выбор» при
запуске обработки открывается плитка из N случайных кадров (по умолчанию
6, диапазон 5–95 % длительности; «Обновить» — новая партия). Кандидаты
извлекаются сразу в финальном качестве (PNG в размере итогового кадра),
выбранный файл уходит в обработку как есть (`image:<путь>`) — без
повторного извлечения и риска соседнего кадра. Партией кандидатов
владеет `VideoService`: предыдущая удаляется при каждом новом запросе.
Без человека (будущая автоматика) режим откатывается к случайному
кадру из 5–95 %.

## Интерфейс

Страница «Видео»: пресеты (карточки + диалог со всеми параметрами; поля
заставки активны только при включённом переключателе «Заставка») и блок
подготовки: исходник → пресет → «Обработать» → **полоса прогресса
с процентами** (ffmpeg отчитывается через `-progress pipe:1`; кодирование —
в фоновом потоке, окно живо) → карточка результата («Открыть»,
«Показать в папке»).

Окно показа вотермарка задаётся **отступами от краёв ролика**
(`wm_start_offset` — появление через N секунд от начала,
`wm_end_offset` — скрыть за N секунд **до конца**; 0/пусто — без
ограничения): отступы переносимы между роликами разной длины,
в абсолютное окно они превращаются конвейером по длительности из
ffprobe. Отступы, не помещающиеся в ролик, — понятная ошибка
до кодирования.

Плавность появления/исчезания (`wm_fade`, сек; 0 — резко): `fade`
по альфа-каналу на краю окна с ненулевым отступом. Статичной картинке
эффект по времени недоступен (кадр один), поэтому при плавности она
зацикливается фильтром `loop` в поток, обязательно ограниченный
`trim` по длительности итога — бесконечный вторичный вход `overlay`
не даёт ffmpeg завершиться. Переходы, не помещающиеся в окно, —
ошибка до кодирования.
