# Модуль «Планировщик»

> Статус: 🟡 черновик-заглушка · Обновлено: 2026-06-30 · Владелец: olegosx

## Назначение

Публиковать посты в заданное время (отложенный постинг). Работает в фоновом
потоке движка, пока запущено приложение; посты, пропущенные за время простоя,
догоняются при следующем старте ([ADR-0006](../02-architecture/decisions/0006-desktop-pyside6.md)).
Своё расписание — основной механизм: Bot API не умеет планировать отправку.
Для постов, идущих через MTProto, доступно серверное планирование
(`schedule_date`) как опция — тогда пост уходит даже при выключенном
приложении ([ADR-0007](../02-architecture/decisions/0007-hybrid-botapi-mtproto.md)).

## Ответственность (план)

- [ ] Хранение запланированных публикаций (что, куда, когда).
- [ ] Срабатывание в нужное время и вызов «Шлюза Telegram».
- [ ] Учёт часовых поясов канала/пользователя.
- [ ] Повтор при сбое публикации, отметка статуса (запланирован/опубликован/ошибка).
