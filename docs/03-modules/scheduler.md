# Модуль «Планировщик»

> Статус: 🟡 черновик-заглушка · Обновлено: 2026-06-30 · Владелец: olegosx

## Назначение

⚠️ **Роль пересмотрена (ADR-0010):** отложенные посты планирует и публикует
**сервер Telegram** (`schedule_date` через userbot) — наш планировщик и
«догон» постам больше не нужны. APScheduler остаётся в движке для будущих
регулярных задач: парсинг источников, периодические проверки.

## Ответственность (план)

- [ ] Хранение запланированных публикаций (что, куда, когда).
- [ ] Срабатывание в нужное время и вызов «Шлюза Telegram».
- [ ] Учёт часовых поясов канала/пользователя.
- [ ] Повтор при сбое публикации, отметка статуса (запланирован/опубликован/ошибка).

## Открытые вопросы

- [x] Механизм: **APScheduler 3.x** ([ADR-0004](../02-architecture/decisions/0004-apscheduler-tasks.md)).
- [x] Поведение при пропущенном времени (приложение было закрыто):
      **догоняем — публикуем пропущенное при следующем запуске**
      ([ADR-0006](../02-architecture/decisions/0006-desktop-pyside6.md)).
- [ ] Повторяющиеся публикации (по расписанию-шаблону) — в рамках или нет?
- [ ] Как помечать «протухшие» пропуски: публиковать всё подряд или есть
      порог опоздания, после которого пост не отправляем?
