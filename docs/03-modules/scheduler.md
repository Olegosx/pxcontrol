# Модуль «Расписание» (отложенные записи)

> Статус: 🟢 реализовано (чтение) · Обновлено: 2026-07-14 · Владелец: olegosx

## Назначение

Показ отложенных записей всех активных каналов. **Своего планировщика
в приложении нет** ([ADR-0010](../02-architecture/decisions/0010-posts-fire-and-forget.md)):
отложенную запись создаёт публикация (`schedule_date` через userbot),
а хранит и публикует её **сервер Telegram** — приложение и компьютер
могут быть выключены.

## Как работает

- Страница «Расписание» читает отложенные записи из Telegram
  (`getScheduledHistory` по каждому активному каналу): источник истины —
  сам канал, локальной таблицы постов нет.
- Править и удалять отложенные можно из любого клиента Telegram.
- Создание постов — на странице «Публикация»
  ([ADR-0011](../02-architecture/decisions/0011-publish-via-userbot.md)).

## Будущее

APScheduler ([ADR-0004](../02-architecture/decisions/0004-apscheduler-tasks.md))
вернётся в проект вместе с регулярными задачами парсинга источников —
к публикации постов он отношения больше не имеет.
