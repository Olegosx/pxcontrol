# Модуль «Каналы»

> Статус: 🟢 реализовано (в объёме текущих функций) · Обновлено: 2026-07-15 · Владелец: olegosx

## Назначение

Подключение Telegram-каналов к приложению и учёт способов их
администрирования. Канал — центральная сущность: на него ссылаются
публикации, пресеты видео и шаблоны подписей.

## Способы подключения (и администрирования)

Канал можно администрировать **ботом**, **userbot-ом** или обоими;
при подключении («Каналы» → «Подключить канал») выбирается способ:

- **Через userbot** (по умолчанию): аккаунт MTProto — админ канала
  с правом публиковать; бот не нужен. Проверка — `get_permissions`
  через Telethon (владельцу канала отдельные права не нужны).
- **Через бота**: бот — админ с правом публиковать; проверка через
  Bot API (`getChat` + `getChatMember`). Попутно опрашивается userbot:
  если он тоже админ — канал помечается обоими способами (для
  публикаций userbot всё равно необходим, ADR-0011).

Карточка канала показывает способы: «админ: userbot»,
«админ: бот <имя>» или «админ: бот <имя> + userbot».

Действия на карточке:

- **Переключатель активности** — настройка `enabled` канала (ADR-0013):
  выключенный канал не показывается в списке выбора на «Публикации»
  и не опрашивается «Расписанием»; сам канал и его данные не трогаются.
- **«Проверить доступы»** — перепроверяет оба способа: флаг userbot
  обновляется в обе стороны (права могли отозвать); потеря прав бота
  его не отвязывает — только предупреждение (бота могут вернуть).
- **«Пресет…»** — пресет обработки видео по умолчанию (настройка
  `default_video_preset`): на странице «Видео» выбор этого канала
  подставляет пресет в панель параметров.
- **«Назначить бота…»** (у канала без бота) — выбор бота с проверкой
  его прав в канале; **«Отвязать бота»** — снимает связь (сам бот
  остаётся в приложении).

## Ввод ссылки

`@имя`, `t.me/…`, `t.me/c/<id>/…` или ID `-100…` (приватные каналы —
только ID). Разбор общий для обоих способов (`normalize_chat_ref`);
пригласительные ссылки `t.me/+…` отклоняются с пояснением. Понятные
ошибки: «канал не найден», «не администратор», «нет права публиковать».

## Решено

- Канал идентифицируется по `tg_chat_id` (числовой ID, уникален);
  @имя хранится справочно.
- Ботов несколько, канал ссылается на своего; `userbot_admin` —
  флаг userbot-администрирования (миграция `f4e8b3c92a17`).
- `video_preset_id` — задел под пресет видео по умолчанию.
- Мультипользовательский доступ не нужен — личный инструмент (vision).
