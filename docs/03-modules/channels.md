# Модуль «Каналы»

> Статус: 🟡 черновик-заглушка · Обновлено: 2026-06-30 · Владелец: olegosx

## Назначение

Подключение Telegram-каналов к приложению, хранение их настроек и проверка
прав бота (бот должен быть администратором с правом публикации).

## Ответственность

- [x] Добавление/удаление канала (`ChannelsService`) — реализовано.
- [x] Проверка при подключении: канал существует, бот — администратор
      с правом публиковать (`check_channel`: getChat + getChatMember).
      Понятные ошибки: «канал не найден», «бот не добавлен», «нет права».
- [x] Принимаемые форматы: `@имя`, ссылка `t.me/…`, числовой ID
      (для приватных каналов) — нормализация в `normalize_chat_ref`.
- [ ] Хранение настроек канала (шаблон видео, расписание по умолчанию).
- [ ] Привязка каналов к источникам и шаблонам генерации.

## Решено

- Канал идентифицируется по `tg_chat_id` (числовой ID из Telegram, уникален);
  @имя хранится справочно.
- Ботов несколько, канал ссылается на своего (ADR-0009).
- Мультипользовательский доступ не нужен — личный инструмент (vision).
