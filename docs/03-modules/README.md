# Модули — карта

> Статус: 🟡 черновик · Обновлено: 2026-07-15 · Владелец: olegosx

## Назначение документа

Перечислить функциональные модули приложения и дать к каждому короткое
описание и ссылку. Модуль — это связный блок бизнес-логики с понятной
зоной ответственности.

## Список модулей

| Модуль | Зона ответственности | Документ |
|---|---|---|
| **Подготовка видео** | FullHD, вотермарк, заставка-превью, обложка (порт `makeVideo`); отделена от публикации | [video-processing.md](video-processing.md) |
| Каналы | Подключение и хранение каналов, права, настройки | [channels.md](channels.md) |
| Публикация и расписание | Единый черновик `PostDraft`; маршрутизация по возможностям канала (userbot/бот); отложки хранит и публикует сервер Telegram (своего планировщика нет, ADR-0010) | [scheduler.md](scheduler.md) |
| Подписи к постам | Поля со словарями, шаблоны подписей и имён файлов | [captions.md](captions.md) |
| Шлюз Telegram | Единая обёртка над Bot API и MTProto: постинг, чтение каналов, лимиты | [telegram-gateway.md](telegram-gateway.md) |
| Источники контента | Парсинг каналов/сайтов/RSS, нормализация | [content-sources.md](content-sources.md) |

## Принцип

Модули общаются через явные интерфейсы, не лезут во внутренности друг
друга. Публикация в Telegram идёт **только** через «Шлюз Telegram» —
остальные модули не вызывают Bot API напрямую.
