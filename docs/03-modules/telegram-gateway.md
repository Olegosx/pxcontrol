# Модуль «Шлюз Telegram»

> Статус: 🟡 черновик-заглушка · Обновлено: 2026-06-30 · Владелец: olegosx

## Назначение

Единая точка взаимодействия с Telegram **поверх двух транспортов** — Bot API
и MTProto ([ADR-0007](../02-architecture/decisions/0007-hybrid-botapi-mtproto.md)).
Весь остальной код обращается к Телеграму только через этот модуль и не знает,
какой транспорт использован.

## Два транспорта

- **Bot API** (`aiogram`) — постинг и управление каналами. Безопасно для аккаунта.
- **MTProto** (`Telethon`, отдельный аккаунт) — чтение/парсинг каналов,
  опционально серверное планирование (`schedule_date`).

Жёсткой границы нет: транспорт выбирается по ситуации. Ориентир — постинг через
Bot API, чтение через MTProto.

## Ответственность (план)

- [x] Проверка токена бота (`getMe`) при добавлении — реализовано.
- [x] Пошаговый вход userbot: код → (пароль 2FA) → строка сессии,
      `MtprotoLoginManager` — реализовано; ошибки Telethon переводятся
      в понятные сообщения (неверный код, FloodWait и т.д.).
- [ ] Отправка постов (текст, медиа, кнопки, медиагруппы) — обычно Bot API.
- [ ] Чтение истории и постов каналов-источников — MTProto.
- [ ] Проверка прав в канале.
- [ ] Обработка ошибок обоих транспортов (нет прав, канал недоступен,
      `429 Too Many Requests`) с повторными попытками и учётом лимитов.
- [ ] Безопасное хранение строки сессии MTProto (см.
      [configuration.md](../06-config/configuration.md)).
