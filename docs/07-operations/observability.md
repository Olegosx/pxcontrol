# Наблюдаемость (логи, метрики, мониторинг)

> Статус: 🟡 черновик-заглушка · Обновлено: 2026-06-30 · Владелец: olegosx

## Назначение документа

Описать, как мы видим состояние работающего приложения: логи, метрики,
оповещения. Это критично, потому что часть работы идёт в фоне без участия
человека — без наблюдаемости незаметно «отвалившийся» парсер или
непрошедшая публикация.

## Реализовано

- **Файл логов с ротацией:** `logs/pxcontrol.log` в папке приложения;
  ротация по размеру (~2 МБ × 5 файлов), кодировка UTF-8. Уровень —
  переменной `LOG_LEVEL` (по умолчанию `INFO`). Параллельно лог идёт
  в консоль.
- **Ошибки операций движка** (всё, что уходит в интерфейс через мост
  `run_in_engine`) пишутся в лог с полным трейсбеком — диагностика
  ведётся по логу.

## Что отслеживать (план)

- [x] Логи через модуль `logging` (не `print`) — файл с ротацией.
- [ ] Статусы публикаций (успех/ошибка), особенно ошибки Bot API.
- [ ] Состояние фоновых задач (расписание, парсинг, генерация).
- [ ] Лимиты Telegram (`429`) и их частота.
- [ ] Расход на вызовы ИИ (стоимость/токены).
