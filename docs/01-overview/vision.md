# Видение и цели

> Статус: 🟢 наполнено (уточняется) · Обновлено: 2026-06-30 · Владелец: olegosx

## Назначение документа

Зафиксировать, **зачем** существует `pXcontrol`, какую боль он снимает
и для кого предназначен. Это верхнеуровневый ориентир: любое решение
по продукту и архитектуре сверяется с тем, что записано здесь.

## Проблема

Главная боль сейчас — **подготовка видео под каждый пост занимает много
времени**. Большая часть постов в каналах — это видео, и перед публикацией
каждый ролик приходится вручную обрабатывать: приводить к единому формату,
ставить водяной знак (вотермарк), готовить кадр для красивого превью
в Телеграме. Это рутина, которая повторяется на каждый пост и съедает
основное время.

Постинг, планирование и поиск тем — вторичные боли по сравнению с монтажом
видео, но тоже отнимают силы при ведении нескольких каналов сразу.

## Что уже сделано

Обработку видео мы начали автоматизировать отдельной утилитой
**`makeVideo`** (соседний проект `../../../makeVideo/`,
[спецификация](../../../makeVideo/SPEC.md)). Это командная утилита на
Python поверх `ffmpeg`, которая за один проход:

1. приводит видео к **FullHD** (вписывает в 1920×1080 без полей);
2. накладывает **вотермарк** (PNG) в заданный угол;
3. делает **заставку** — держит выбранный кадр секунду и плавно растворяет
   его в видео, чтобы в превью Телеграма попадал красивый кадр, а не чёрный
   экран или титры;
4. при желании вшивает **обложку** в `mp4` для надёжного превью.

`makeVideo` — отдельный проект и наш **референс** по обработке видео,
а не внешняя зависимость. Нужные наработки (логику работы с `ffmpeg`)
переносим прямо в код `pXcontrol` — в модуль обработки видео
(см. раздел «Главная ставка»).

## Целевая аудитория

- **Личный инструмент.** Пользователь — владелец каналов (вы), ведёт их сам.
  Не публичный сервис (SaaS), без внешних клиентов на старте.
- **Масштаб — до ~10 каналов.** Один оператор, средний поток постов.

Это определяет технические решения: **десктопное приложение** (видео
локальное и тяжёлое — гонять по сети нерационально), упор на простоту
(KISS/YAGNI), SQLite как локальная база, нативный интерфейс на PySide6
(см. [технологический стек](../02-architecture/tech-stack.md) и
[ADR-0006](../02-architecture/decisions/0006-desktop-pyside6.md)).

## Цели продукта

1. **Сократить время на видео-пост** — убрать ручной монтаж: загрузил
   ролик → получил готовый к публикации файл с единым брендингом и превью.
2. **Единый вид и брендинг** всех каналов без ручной возни (вотермарк,
   формат, превью — по шаблону канала).
3. **Регулярность публикаций** на ~10 каналах из одного места —
   отложенный постинг по расписанию.
4. **Снять вторичную рутину** — генерация тем/текстов ИИ и сбор контента
   из источников (это поддержка, а не ядро).

## Главная ставка

`pXcontrol` строится вокруг **видео-центричного конвейера поста**:

```
видео-исходник → обработка (makeVideo: FullHD, вотермарк, заставка, обложка)
   → текст поста (вручную или ИИ) → канал и время → публикация по расписанию
```

Обработка видео — не побочная функция, а сердце продукта. Логику из
`makeVideo` переносим в **модуль обработки видео** внутри проекта: вызов
`ffmpeg` с параметрами по шаблону канала (какой вотермарк, какие настройки
заставки).

## Границы (что НЕ делаем)

- Telegram — гибрид **Bot API + MTProto**: постинг через бота, чтение/парсинг
  каналов через userbot на отдельном аккаунте
  ([ADR-0007](../02-architecture/decisions/0007-hybrid-botapi-mtproto.md)).
- **Не SaaS** на старте — личный инструмент, без биллинга и мультиарендности.
- Не редактор видео общего назначения: `makeVideo` делает фиксированный
  конвейер под Телеграм, а не произвольный монтаж.

## Метрики успеха (предварительно, уточнить)

- [ ] Время на подготовку одного видео-поста: с «N минут ручной работы»
      до «загрузил и подтвердил».
- [ ] Доля постов, опубликованных по расписанию без ручного вмешательства.

## Открытые вопросы

- [ ] Тематика и язык контента каналов (влияет на шаблоны генерации ИИ).
- [ ] Как сейчас хранятся исходники видео и куда складывать обработанные.
- [ ] Нужна ли пакетная обработка нескольких видео сразу.
