## Термины (заполнять по мере появления)

| Термин | Расшифровка / определение |
|---|---|
| **Bot API** | Официальный HTTP-интерфейс Telegram для ботов. Через него приложение публикует посты в каналы. |
| **MTProto** | Низкоуровневый протокол Telegram для клиентов (userbot). Используется как второй транспорт наряду с Bot API — для чтения/парсинга каналов и опц. планирования (см. ADR-0007). Работает на отдельном аккаунте. |
| **userbot** | Автоматизация от имени пользовательского аккаунта Telegram (через MTProto), а не бота. Даёт чтение чужих каналов, но несёт риск блокировки аккаунта. |
| **session string** | Строка сессии MTProto — по силе равна паролю аккаунта; хранится зашифрованно, не попадает в код и логи. |
| **Канал** | Telegram-канал, которым управляет приложение; бот — его администратор. |
| **Пост** | Единица контента для публикации (текст, медиа, кнопки). |
| **Источник** | Внешний поставщик контента: другой канал, сайт или RSS-лента. |
| **Очередь модерации** | Список заготовок контента, ожидающих одобрения человеком. |
| **Планировщик** | Компонент, публикующий посты в заданное время. |
| **Отложенный постинг** | Публикация поста в будущем по расписанию, а не сразу. |
| **Рерайт** | Переписывание исходного материала перед публикацией (часто с помощью ИИ). |
| **ADR** | Architecture Decision Record — запись о принятом архитектурном решении. |
