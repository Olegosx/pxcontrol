# Стандарты кода

> Статус: 🟡 черновик-заглушка · Обновлено: 2026-06-30 · Владелец: olegosx

## Назначение документа

Зафиксировать правила оформления кода для `pXcontrol`. Базовые стандарты
заданы глобально в личном `~/.claude/CLAUDE.md` и имеют приоритет; здесь —
их краткая выжимка и проектные дополнения.

## Базовые правила (из глобального стандарта)

- Python 3.x, **строгая типизация** (type hints везде).
- Имена: `snake_case` для переменных/функций, `PascalCase` для классов.
- Docstrings (Google-style) для всех публичных функций и классов.
- Максимум **30 строк** на функцию; длиннее — рефакторить.
- Принципы DRY, KISS, YAGNI.
- Явная обработка ошибок — никогда не глотать исключения молча.
- Логирование через `logging`, не `print`. f-строки для форматирования.
- Отступы — табы (где синтаксис позволяет).

## Проектные дополнения (заполнить)

- [ ] Линтеры/форматтеры (ruff, mypy и т.п.) и их конфигурация.
- [ ] Структура пакетов проекта.
- [ ] Соглашения по слоям (модули не вызывают Bot API в обход шлюза).

## Git

- Commit messages: `type(scope): описание` (Conventional Commits).
- Ветки: `type/short-description`. Не коммитить напрямую в `main`.
