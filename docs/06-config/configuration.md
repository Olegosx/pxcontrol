# Конфигурация

> Статус: 🟢 актуально · Обновлено: 2026-07-16 · Владелец: olegosx

## Назначение документа

Описать, как настраивается приложение и где что хранится (ADR-0009).

## Правило выбора хранилища (ADR-0013)

| Вид данных | Признак | Хранилище |
|---|---|---|
| Настройка приложения | одно значение на всё приложение, без связей | `app_settings` через реестр ключей |
| Настройка сущности | принадлежит каналу/учётке/пресету | строка в `<сущность>_settings` (FK, каскад) или колонка сущности, если это идентичность/связь |
| Секрет | токен, сессия, ключ API | шифрованная колонка своей сущности (никогда — в настройки) |
| Бутстрап | нужно до открытия БД | `.env` |

## Уровни конфигурации

### 1. Бутстрап — файл `.env` (только то, что нужно ДО открытия БД)

| Переменная | Назначение | По умолчанию |
|---|---|---|
| `DATABASE_URL` | путь к локальной БД | файл `pxcontrol.db` **в папке приложения** |
| `LOG_LEVEL` | уровень логирования | `INFO` |
| `FFMPEG_PATH` | запасное значение пути к `ffmpeg`: действует, пока настройка в приложении пуста | `ffmpeg` |

Шаблон — `.env.example`; сам `.env` в git не попадает.

**Портативный режим (решение от 06.07.2026):** БД и `.env` ищутся
в **каталоге приложения** (в разработке — корень проекта, в собранном
виде — папка исполняемого файла), а не в каталоге запуска. Перенос папки
приложения переносит и все данные. Оговорка: каталог приложения должен
быть доступен на запись (портативная установка, не системная).

### 2. Всё остальное — в БД, управляется из интерфейса

- **Настройки приложения** — таблица `app_settings` (строка
  «имя → значение»); состав, типы и умолчания задаёт **реестр ключей**
  (`engine/services/settings.py`, ADR-0013). Сейчас: тема оформления,
  состояние окна, последний канал «Публикации», путь к `ffmpeg`
  (страница «Настройки»; пусто — бутстрап из `.env`; смена
  подхватывается без перезапуска) и **папки видео** («Настройки →
  Папки»): `video_source_dir` (исходники), `video_processed_dir`
  (результаты обработки), `video_published_dir` (опубликованные);
  пусто — `media/source|processed|published` в папке приложения.
  Внутри каждой — подпапка пресета (`video_presets.subdir`); после
  публикации видео переезжает из результатов в опубликованные
  с сохранением подпапки.
- **Настройки канала** — таблица `channel_settings` (строка
  «(канал, имя) → значение», внешний ключ с каскадом); тот же реестр
  ключей. Ключи: `enabled` (канал активен — переключатель в карточке;
  выключенный не показывается на «Публикации» и не опрашивается
  «Расписанием») и `default_video_preset` (пресет обработки
  по умолчанию — кнопка «Пресет…» в карточке; на странице «Видео»
  выбор канала подставляет его пресет в панель).
- Боты, userbot-аккаунты, ключи ИИ — таблицы `bots` / `tg_accounts` /
  `ai_credentials`; секретные поля **шифруются**.
- Шаблоны видео и каналы — `video_presets` / `channels`.

Подробно — [модель данных](../05-data/data-model.md).

## Безопасность секретов (ADR-0009)

- Секреты шифруются схемой Fernet (`cryptography`); в файле БД — только
  шифртекст.
- Ключ шифрования живёт в **системном хранилище ключей ОС** (`keyring`:
  GNOME Keyring/KWallet) и создаётся автоматически при первом запуске.
- Требование: работающее системное хранилище. Без него приложение при
  первом обращении к секретам честно сообщает об ошибке.
- Потеря ключа = секреты не восстановить, вводятся заново.
- Секреты никогда не пишутся в логи и не попадают в код.
