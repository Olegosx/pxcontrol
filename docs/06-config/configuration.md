# Конфигурация

> Статус: 🟢 актуально · Обновлено: 2026-07-03 · Владелец: olegosx

## Назначение документа

Описать, как настраивается приложение и где что хранится (ADR-0009).

## Два уровня конфигурации

### 1. Бутстрап — файл `.env` (только то, что нужно ДО открытия БД)

| Переменная | Назначение | По умолчанию |
|---|---|---|
| `DATABASE_URL` | путь к локальной БД | файл `pxcontrol.db` **в папке приложения** |
| `LOG_LEVEL` | уровень логирования | `INFO` |

(`FFMPEG_PATH` вернётся с шагом обработки видео; `TZ` удалён — время
постов задаётся в локальном времени и хранится сервером Telegram.)

Шаблон — `.env.example`; сам `.env` в git не попадает.

**Портативный режим (решение от 06.07.2026):** БД и `.env` ищутся
в **каталоге приложения** (в разработке — корень проекта, в собранном
виде — папка исполняемого файла), а не в каталоге запуска. Перенос папки
приложения переносит и все данные. Оговорка: каталог приложения должен
быть доступен на запись (портативная установка, не системная).

### 2. Всё остальное — в БД, управляется из интерфейса

- Настройки приложения — таблица `settings` (ключ → JSON-значение).
- Боты, userbot-аккаунты, ключи ИИ — таблицы `bots` / `tg_accounts` /
  `ai_credentials`; секретные поля **шифруются**.
- Шаблоны видео и каналы — `video_presets` / `channels`.

Подробно — [модель данных](../05-data/data-model.md).

## Безопасность секретов (ADR-0009)

- Секреты шифруются схемой Fernet (`cryptography`); в файле БД — только
  шифртекст.
- Ключ шифрования живёт в **системном хранилище ключей ОС** (`keyring`:
  GNOME Keyring/KWallet) и создаётся автоматически при первом запуске.
- Требование: работающее системное хранилище. Без него приложение при
  первом обращении к секретам честно сообщает об ошибке.
- Потеря ключа = секреты не восстановить, вводятся заново.
- Секреты никогда не пишутся в логи и не попадают в код.
