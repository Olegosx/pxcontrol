# Порядок запуска и остановки

> Статус: 🟢 реализовано в каркасе · Обновлено: 2026-07-01 · Владелец: olegosx

## Назначение документа

Описать, как приложение запускается и останавливается, и как связаны
интерфейс и движок во время работы. Соответствует коду в `src/pxcontrol/`.

## Модель потоков

- **Главный поток** — интерфейс на PySide6 и цикл событий Qt.
- **Фоновый поток `engine`** — движок со своим циклом событий asyncio:
  база данных, шлюз Telegram, планировщик, обработка видео.

Так тяжёлая и асинхронная работа не блокирует окно (см.
[ADR-0006](decisions/0006-desktop-pyside6.md)). Интерфейс вызывает корутины
движка через `EngineWorker.submit(coro)` — она безопасно планирует корутину
в цикле движка из потока интерфейса и возвращает `Future`.

## Порядок запуска

Реализован в `app.run()` и `EngineWorker`:

1. **Настройки** — `get_settings()` читает переменные окружения и `.env`.
2. **Логирование** — `setup_logging(level)`.
3. **Движок в фоновом потоке** — `EngineWorker.start()` поднимает поток
   `engine`, создаёт цикл asyncio и выполняет `Engine.start()`:
   1. инициализация базы данных — миграции Alembic (`Database.init()`);
   2. активация userbot, если в БД есть аккаунт с сессией;
   3. запуск шлюза Telegram (подключение userbot; Bot API соединений
      не держит — боты работают по токену на операцию).
   `start()` ждёт готовности движка и пробрасывает ошибку запуска, если она была.

   Отдельного планировщика нет: отложенные посты хранит и публикует
   сервер Telegram (ADR-0010), «догон» не требуется.
4. **Интерфейс** — создаётся `QApplication`, показывается `MainWindow`.
5. **Цикл событий Qt** — `app.exec()` (блокирует главный поток до закрытия окна).

## Порядок остановки (обратный)

При выходе из `app.run()` вызывается `EngineWorker.stop()`:

1. цикл движка останавливается (`loop.stop()` из главного потока);
2. выполняется `Engine.stop()` — в обратном порядке:
   шлюз Telegram (`stop`) → база данных (`close`);
3. фоновый поток завершается (`join`).

## Headless-режим (без интерфейса)

`app.run_headless(seconds)` запускает только движок — для проверки и тестов
без графики. На нём держится дымовой тест `tests/test_smoke.py`.
