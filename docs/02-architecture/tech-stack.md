# Технологический стек

> Статус: 🟢 ключевые решения приняты, детали уточняются · Обновлено: 2026-06-30 · Владелец: olegosx

## Назначение документа

Зафиксировать выбранные технологии и **обоснование** каждого выбора.
Любая технология добавляется сюда вместе с причиной «почему именно она».

## Тип приложения

**Десктопное приложение** (см. [ADR-0006](decisions/0006-desktop-pyside6.md)).
Архитектура: UI-независимый **движок** (engine) — отдельный Python-пакет
с бизнес-логикой — и тонкий **интерфейс** на PySide6 поверх него. Веб-сервер
не используется.

## Принятые решения (ядро)

| Слой | Технология | Обоснование |
|---|---|---|
| Язык | **Python 3.x**, строгая типизация | Стандарт проекта (личный `CLAUDE.md`). |
| Telegram | **Bot API + MTProto (гибрид)** | [ADR-0007](decisions/0007-hybrid-botapi-mtproto.md). Транспорт выбирается по ситуации. |
| Архитектура | **Движок (Python-пакет) + тонкий UI** | Логика отделена от интерфейса, вызывается напрямую. [ADR-0006](decisions/0006-desktop-pyside6.md). |
| Десктопный UI | **PySide6 (Qt)** | Нативное окно и видео-плеер (`QtMultimedia`) для просмотра видео. [ADR-0006](decisions/0006-desktop-pyside6.md). |
| Компоненты UI | **QFluentWidgets** (PySide6-Fluent-Widgets) | Готовые компоненты Fluent: боковая навигация, темы, карточки. ⚠️ GPLv3 для некоммерческого использования. [ADR-0008](decisions/0008-qfluentwidgets.md). |
| Библиотека Bot API | **aiogram 3.x** | Типизированный async-клиент Bot API (постинг, управление). В десктопе движок работает в фоновом потоке со своим циклом событий. |
| Библиотека MTProto | **Telethon** | Зрелый async-клиент userbot для чтения/парсинга каналов и опц. серверного планирования. Работает на отдельном аккаунте. [ADR-0007](decisions/0007-hybrid-botapi-mtproto.md). |
| ORM | **SQLAlchemy 2.0 (async)** | Стандарт, типизирован, абстрагирует СУБД. |
| Миграции | **Alembic** (подключён) | Миграции применяются при старте (`Database.init()`); ручной запуск — `alembic.ini` в корне. |
| СУБД | **SQLite** (локальный файл) | Один пользователь, локальное хранение — очевидный выбор для десктопа. Конфиг и секреты — тоже в БД ([ADR-0009](decisions/0009-config-secrets-in-db.md)). |
| Секреты | **cryptography (Fernet) + keyring** | Секретные поля БД шифруются; ключ — в системном хранилище ОС. [ADR-0009](decisions/0009-config-secrets-in-db.md). |
| Отложенные посты | **Сервер Telegram** (`schedule_date` через userbot) | Публикует Telegram, приложение может быть выключено. [ADR-0010](decisions/0010-posts-fire-and-forget.md). APScheduler убран из зависимостей — вернётся с парсингом источников ([ADR-0004](decisions/0004-apscheduler-tasks.md)). |
| Обработка видео | **`ffmpeg` напрямую** | Ядро продукта (FullHD, вотермарк, превью). Наработки втягиваются в проект из соседнего `makeVideo` как **референс**, не как внешняя зависимость. |
| HTTP-клиент | **httpx** | Запросы к ИИ-провайдеру и источникам. |
| Парсинг RSS | **feedparser** | RSS — самый простой источник, первый по дорожной карте. |
| Парсинг HTML | **selectolax** или BeautifulSoup (позже) | Для сайтов-источников на следующем этапе. |
| Провайдер ИИ | **Claude API** (SDK `anthropic`) | Рекомендуется по окружению; качественная генерация текста. |
| Конфигурация | **pydantic-settings** | Чтение настроек из переменных окружения с валидацией. |
| Тесты | **pytest + pytest-asyncio + respx** | Стандарт; `respx` мокает HTTP-вызовы `httpx`. |
| Линтер/форматтер | **ruff** | Быстрый линт и форматирование в одном инструменте. |
| Проверка типов | **mypy** (strict) | Под мандат строгой типизации. |
| Менеджер окружения | **uv** | Быстрая установка и блокировка зависимостей. |

## Важно: обработка видео и планирование

- **Видео — тяжёлая фоновая задача.** Кодирование через `ffmpeg` идёт минуты
  и грузит процессор, поэтому выполняется **только в фоновом потоке движка**,
  не в потоке интерфейса (иначе окно зависнет). На личном масштабе (≤10
  каналов) последовательная обработка приемлема.
- **Расписание постов — на стороне Telegram (ADR-0010).** Bot API не умеет
  отложенную отправку, зато MTProto умеет `schedule_date`: отложенная
  запись создаётся прямо в канале, публикует её сервер Telegram. Свой
  планировщик приложению для постов не нужен.

## Исключено из стека (после перехода на десктоп)

- **FastAPI** — веб-сервер не нужен, движок вызывается напрямую
  ([ADR-0003](decisions/0003-fastapi-backend.md) → superseded). Вернётся
  только если понадобится удалённое управление.
- **Jinja2 + HTMX** — веб-фронтенд отменён, интерфейс нативный на PySide6
  ([ADR-0005](decisions/0005-frontend-htmx.md) → superseded).
- **`makeVideo` как внешняя CLI-утилита** — используем его только как
  референс, нужные части переносим в код проекта.

## Отклонённые альтернативы (кратко)

- **Веб-приложение** вместо десктопа — заливка тяжёлого видео по сети,
  хостинг и веб-безопасность ради ненужного удалённого доступа
  ([ADR-0006](decisions/0006-desktop-pyside6.md)).
- **Другие десктоп-UI** (`tkinter`, `Flet`, `pywebview`) — слабее по показу
  видео или младше по экосистеме, чем PySide6.
- **Celery + Redis** вместо APScheduler — лишний брокер и инфраструктура,
  не нужно на старте ([ADR-0004](decisions/0004-apscheduler-tasks.md)).
