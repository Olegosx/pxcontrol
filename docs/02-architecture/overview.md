# Обзор архитектуры

## Назначение документа

Дать общую картину системы «с высоты птичьего полёта»: какие крупные
части есть, как они связаны, где проходят границы. Детали — в отдельных
документах по компонентам и модулям.

## Контекст (заполнить)

Кто и что взаимодействует с системой снаружи:

- **Пользователь** → десктопное приложение (нативное окно).
- **Приложение** → Telegram Bot API (публикация постов).
- **Приложение** → внешние источники (сайты, RSS, каналы) для парсинга.
- **Приложение** → провайдер ИИ (например, Claude API) для генерации.

## Крупные части системы (предварительно)

1. **Интерфейс (PySide6)** — нативное окно приложения, в том числе просмотр видео.
2. **Движок (engine)** — UI-независимый Python-пакет с бизнес-логикой;
   интерфейс вызывает его напрямую в процессе.
3. **Фоновый поток движка** — обработка видео, публикация по расписанию,
   парсинг источников, генерация ИИ; вынесен в поток, чтобы окно не зависало.
4. **Хранилище** — SQLite (каналы, посты, расписание, очередь, источники).

> ⚠️ Тяжёлая и фоновая работа (обработка видео через `ffmpeg`, расписание)
> идёт в отдельном потоке движка, а не в потоке интерфейса. Планировщик
> работает, пока приложение открыто, и догоняет пропущенное при следующем
> старте (см. [ADR-0006](decisions/0006-desktop-pyside6.md)).
