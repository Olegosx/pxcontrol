# ADR-0009. Конфигурация и секреты — в БД, шифрование через keyring

> Статус: accepted
> Дата: 2026-07-03 · Автор: olegosx

## Контекст

Каналов несколько, у каждого может быть свой бот и свой шаблон видео.
Конфигурация должна редактироваться из интерфейса, а не руками в `.env`.
Секреты (токены ботов, реквизиты и сессия userbot, ключи ИИ) при этом
нельзя хранить в файле SQLite открытым текстом: файл легко попадает
в бэкап/синхронизацию, а сессия userbot равна паролю от аккаунта.

## Решение

1. **Конфигурация и секреты живут в БД** и управляются из интерфейса.
   В `.env` остаётся только бутстрап (то, что нужно до открытия БД):
   `DATABASE_URL`, `FFMPEG_PATH`, `TZ`, `LOG_LEVEL`.
2. **Секретные поля шифруются** схемой Fernet (библиотека `cryptography`).
   Ключ шифрования хранится в **системном хранилище ключей ОС** через
   `keyring` (GNOME Keyring/KWallet). Копия файла БД без доступа к машине
   бесполезна.
3. Шифрование прозрачно для кода: тип колонки `EncryptedStr`
   (`TypeDecorator` SQLAlchemy) шифрует при записи и расшифровывает при
   чтении; сервисы работают с обычными строками.
4. **Ботов может быть несколько** — таблица `bots`, канал ссылается на
   своего бота (`channels.bot_id`).
5. Схема БД версионируется **Alembic**; `Database.init()` применяет
   миграции при старте.

## Альтернативы

- **Секреты открытым текстом в БД** — проще, но любая копия файла = утечка
  всех токенов и аккаунта userbot. Отклонено.
- **Секреты оставить в `.env`** — не масштабируется на несколько
  ботов/аккаунтов и не редактируется из интерфейса. Отклонено.
- **Один общий бот в настройках** — при желании развести каналы по ботам
  потребовалась бы миграция схемы. Отклонено в пользу таблицы `bots` сразу.

## Последствия

- **Плюсы:** управление из GUI; несколько ботов/аккаунтов; файл БД
  безопасен вне машины; схема версионируется.
- **Минусы / риски:**
  - Требуется работающее системное хранилище ключей. На десктопе с
    GNOME/KDE есть всегда; на «голой» системе приложение честно упадёт
    при первом обращении к секретам с понятным сообщением.
  - Потеря ключа из хранилища = секреты не расшифровать (нужно будет
    ввести их заново). Ключ создаётся автоматически при первом запуске.
- **Контентные таблицы** (posts, media, sources, queue_items,
  generation_jobs) сознательно отложены (YAGNI) — добавляются миграциями
  вместе с соответствующими функциями.
