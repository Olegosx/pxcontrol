## Контекст

Нужен веб-фреймворк для бэкенда: REST API для веб-панели плюс асинхронная
работа с внешними сервисами (Telegram Bot API, провайдер ИИ, парсинг
источников). Стандарт проекта — Python со строгой типизацией.

## Решение

Используем **FastAPI**.

## Альтернативы

- **Django (+ DRF).**
  - Плюсы: «батарейки в комплекте», админка, зрелая экосистема.
  - Минусы: тяжелее, ORM-центричен, исторически синхронный; для нашего
    профиля (асинхронный ввод-вывод, чистый API) избыточен.
  - Отклонено.
- **Flask.**
  - Плюсы: минималистичен, гибок.
  - Минусы: слабее история по асинхронности и типизации, больше ручного
    «клея» (валидация, схемы, документация).
  - Отклонено.

## Последствия

- **Плюсы:** асинхронность под профиль нагрузки; Pydantic-валидация
  совпадает с мандатом «type hints везде»; автогенерация OpenAPI/Swagger
  — даром закрывает часть [документации API](../../04-api/engine-api.md).
- **Минусы / риски:** меньше «из коробки», чем у Django (нет готовой
  админки) — компенсируется простотой задач.
- **Что дальше:** определить структуру приложения и слой доступа к данным.
