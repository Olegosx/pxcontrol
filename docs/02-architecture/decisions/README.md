# Журнал архитектурных решений (ADR)

> Статус: 🟡 черновик-заглушка · Обновлено: 2026-06-30 · Владелец: olegosx

## Что это

**ADR (Architecture Decision Record)** — короткая запись об одном принятом
архитектурном решении: контекст, само решение, альтернативы и последствия.
Записи не редактируются «задним числом»: если решение пересматривается,
создаётся новая запись, а старая помечается статусом «заменено».

## Статусы записей

- **proposed** — предложено, ещё не утверждено.
- **accepted** — принято и действует.
- **superseded** — заменено более новой записью (с ссылкой на неё).
- **deprecated** — отменено без замены.

## Как добавить запись

1. Скопировать [template.md](template.md) в файл `NNNN-краткое-название.md`
   (номер — следующий по порядку, с ведущими нулями).
2. Заполнить разделы.
3. Добавить строку в реестр выше.
