# ADR-0011. Публикация всего контента — единой сущностью через userbot

> Статус: accepted
> Дата: 2026-07-14 · Автор: olegosx
>
> Уточняет [ADR-0007](0007-hybrid-botapi-mtproto.md)
> и [ADR-0010](0010-posts-fire-and-forget.md) (пункты 2–3 решения).

## Контекст

ADR-0010 разводил публикацию по транспортам: «сейчас» — бот (Bot API),
отложенно — userbot (MTProto). С появлением видео схема поплыла: лимит
Bot API на отправку файлов ботом — 50 МБ
([FAQ Bot API](https://core.telegram.org/bots/faq#how-do-i-upload-a-large-file)),
а обработанные ролики — сотни мегабайт, поэтому видео пришлось отправлять
через userbot даже «сейчас». Получилось два пути публикации: текст так,
видео иначе — зоопарк транспортов и дублирование интерфейса.

## Решение

1. **Единая сущность публикации** — черновик `PostDraft` (канал, текст,
   вложение с типом, время) и один метод `PostsService.publish()`.
   Типы вложений: фото, видео, аудио, документ; текст — тело поста или
   подпись к медиа. Расширение (альбомы, опросы) — новые поля черновика,
   вход тот же.
2. **Единый транспорт публикации — userbot (MTProto)** в обоих режимах:
   «сейчас» — обычная отправка, отложенно — `schedule_date` (сервер
   Telegram хранит и публикует, ADR-0010 п.1 в силе). Лимит MTProto
   на файл — 2 ГБ.
3. **Путь через бота законсервирован, не удалён**: `send_now` (текст
   ботом) остаётся публичным методом движка с тестами — пригодится для
   генерации ИИ. Bot API продолжает обслуживать проверки токена,
   проверку прав в канале и диагностику.
4. Интерфейс: единая страница «Публикация» (тип контента — сегментами);
   «Расписание» — только список отложенных; страница «Видео» передаёт
   готовый файл на «Публикацию» с предзаполнением.

## Альтернативы

- **Оставить два пути (бот для текста, userbot для медиа).** Меньше
  нагрузки на аккаунт userbot, но два кода публикации, две ветки ошибок
  и путаница в интерфейсе. Отклонено.
- **Локальный сервер Bot API (лимит 2 ГБ).** Снимает ограничение файла,
  но добавляет отдельный процесс-зависимость ради того, что MTProto
  умеет из коробки. Отклонено.

## Последствия

- **Плюсы:** один код публикации на все типы контента; одна страница
  в интерфейсе; общий прогресс загрузки; отложки и «сейчас» ведут себя
  одинаково.
- **Минусы/риски:** вся публикация — с пользовательского аккаунта:
  автоматизация userbot — серая зона правил Telegram. Компромисс принят
  осознанно: объёмы личные (до ~10 каналов, ручной запуск), отложенные
  и так шли через userbot, бот-путь сохранён как запасной.
- Требование к каналу: userbot — админ с правом публиковать (раньше
  нужно было только для отложенных). Бот-админ пока остаётся условием
  подключения канала (проверка прав) — пересмотрим отдельно.
