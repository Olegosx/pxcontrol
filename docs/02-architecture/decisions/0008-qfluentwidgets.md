# ADR-0008. Библиотека компонентов интерфейса — QFluentWidgets

> Статус: accepted
> Дата: 2026-07-02 · Автор: olegosx
>
> Дополняет [ADR-0006](0006-desktop-pyside6.md) (PySide6 остаётся основой).

## Контекст

Интерфейс решено развивать «эстетика вперёд»: сначала оболочка и внешний
вид, функции подключаются по одной. Рисовать дизайн-систему с нуля на QSS
(стилевых таблицах Qt) — самый трудозатратный путь. Рассматривались готовые
библиотеки тем и компонентов для PySide6.

## Решение

Используем **PySide6-Fluent-Widgets (QFluentWidgets)** — библиотеку готовых
компонентов в стиле Fluent (дизайн-язык Windows 11): окно с боковой
навигацией (`FluentWindow` — совпадает с утверждённой раскладкой), карточки,
переключатели, уведомления, тёмная/светлая тема одной строкой, настраиваемый
акцентный цвет.

Акцентный цвет по умолчанию — **тиловый `#14b8a6`** (выбран по
интерактивному макету). Тема по умолчанию — тёмная, переключатель
в «Настройках».

## Альтернативы

- **qt-material** — только «покраска» стандартных виджетов под Material,
  компоненты (карточки, навигацию) пришлось бы собирать самим. Отклонено.
- **PyQtDarkTheme** — проект фактически заморожен (требует Python <3.12,
  жив только форк). Отклонено — плохая ставка на будущее.
- **Своя тема на QSS** — полный контроль, но недели работы; для личного
  инструмента не оправдано. Отклонено.

## Последствия

- **Плюсы:** готовая боковая навигация, компоненты и темы — быстрый путь
  к аккуратному интерфейсу; активная поддержка (1.11.x, 2026).
- **Минусы / риски:**
  - **Лицензия GPLv3 для некоммерческого использования.** Для личного
    инструмента приемлемо. Если приложение когда-либо будет
    распространяться/продаваться — потребуется коммерческая лицензия
    библиотеки либо её замена. Риск зафиксирован осознанно.
  - Принимаем дизайн-язык Fluent; кастомизация «в сторону» от него —
    борьба с библиотекой.
- **Что дальше:** оболочка окна на `FluentWindow`, страницы-заглушки
  разделов, переключатель темы в «Настройках».
