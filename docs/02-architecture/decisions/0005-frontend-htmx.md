# ADR-0005. Фронтенд на Jinja2 + HTMX

> Статус: superseded — заменено [ADR-0006](0006-desktop-pyside6.md)
> Дата: 2026-06-30 · Автор: olegosx
>
> ⚠️ Веб-фронтенд отменён вместе с переходом на десктоп. Интерфейс — нативный
> на PySide6 (Qt). См. [ADR-0006](0006-desktop-pyside6.md).

## Контекст

Веб-панель управления должна давать динамичный интерфейс (списки каналов,
расписание, очередь модерации с действиями), но проект — инструмент,
вероятно, для одного оператора или небольшой команды, а не публичный
продукт с богатым UX. Стек бэкенда — Python/FastAPI.

## Решение

На старте делаем **серверный рендеринг через Jinja2-шаблоны + HTMX** —
библиотеку, которая добавляет динамику (подгрузка фрагментов страницы,
действия без полной перезагрузки) обычными HTML-атрибутами, без отдельного
JavaScript-приложения.

## Альтернативы

- **SPA (React/Vue) + отдельный фронтенд.**
  - Плюсы: богатый интерфейс, привычный для продуктовых команд.
  - Минусы: отдельный стек сборки JS, отдельная разработка и поддержка;
    избыточно для инструмента одного оператора.
  - Отклонено на старте.

## Последствия

- **Плюсы:** один язык и один стек (Python + шаблоны), быстрый путь к MVP,
  минимум зависимостей на фронте.
- **Минусы / риски:** при усложнении UX HTMX упирается в потолок —
  тогда оправдан переход на SPA.
- **Условие снято:** [vision.md](../../01-overview/vision.md) подтвердил —
  это личный инструмент на ~10 каналов, не SaaS. Богатый интерфейс не нужен,
  HTMX достаточно. Если в будущем продукт пойдёт на внешних пользователей —
  пересмотреть в пользу SPA новой записью ADR.
