## Контекст

`pXcontrol` — видео-центричный инструмент для одного пользователя
(см. [vision.md](../../01-overview/vision.md)). Ключевые факты:

- Исходники видео — большие локальные файлы (FullHD), их обработка идёт
  локальным `ffmpeg`. Гонять их по сети на сервер и обратно нерационально.
- Пользователь один, удалённый доступ и многопользовательскость не нужны.
- Десктоп у пользователя работает практически круглосуточно.
- **Проверено:** Bot API не умеет планировать отправку нативно
  (`schedule_date` — функция MTProto, не Bot API). Значит, планировщик нужен
  у нас в любом случае — это не аргумент в пользу веб-сервера.

## Решение

Делаем **десктопное приложение**. Архитектурно разделяем:

1. **Движок (engine)** — UI-независимый Python-пакет: модель данных,
   оркестровка обработки видео, планировщик, шлюз Телеграма, бизнес-логика.
2. **Интерфейс** — тонкая оболочка на **PySide6 (Qt)** поверх движка,
   вызывает его напрямую в процессе. Веб-сервер (FastAPI) не нужен.

Планировщик (`APScheduler`) работает внутри приложения; пропущенные за время
простоя публикации **догоняются при следующем запуске** (вариант A из
обсуждения). Движок выполняет асинхронную работу в фоновом потоке со своим
циклом событий; интерфейс общается с ним через сигналы Qt, чтобы окно не
зависало во время обработки видео.

## Альтернативы

- **Веб-приложение (ADR-0002, ADR-0003, ADR-0005).**
  - Минусы: заливка тяжёлого видео по сети, хостинг, HTTPS, аутентификация —
    всё ради ненужного удалённого доступа.
  - Отклонено.
- **Другие десктоп-UI:** `tkinter` (слабый показ видео), `Flet` (молодая
  экосистема), «локальный веб в окне» через `pywebview` (видео и доступ
  к локальным файлам менее нативны). PySide6 даёт штатный видео-виджет
  (`QtMultimedia`) — прямо закрывает потребность в просмотре видео.

## Последствия

- **Плюсы:** видео обрабатывается и хранится локально; нативный просмотр
  видео в окне; нет хостинга и веб-безопасности; SQLite как локальная БД
  становится ещё очевиднее.
- **Минусы / риски:** отложенная публикация срабатывает только пока машина
  включена. Снято тем, что десктоп работает 24/7 **плюс** догон пропущенного
  при старте.
- **Открытый под-вопрос:** способ интеграции асинхронного движка с циклом
  событий Qt (`qasync` либо отдельный поток с собственным циклом) —
  определить при реализации. Возможен и полностью синхронный шлюз Телеграма.
