# Компоненты системы

> Статус: 🟢 актуально · Обновлено: 2026-07-14 · Владелец: olegosx

## Назначение документа

Перечислить компоненты приложения, их зону ответственности и связи между
ними. Уровень детализации — крупнее, чем модули, но конкретнее, чем общий
обзор.

## Состав

### Интерфейс (PySide6 + QFluentWidgets)
- Страницы: «Каналы», «Видео» (пресеты и подготовка), «Публикация»
  (все типы контента + шаблоны подписей), «Расписание» (отложенные
  из Telegram), «Настройки → Аккаунты».
- К движку обращается только через `EngineWorker.submit()` +
  мост `run_in_engine` (результаты возвращаются сигналами Qt).
- Общие элементы страниц — `pages/common.py`: карточки-строки, диалоги,
  панель прогресса (`ProgressPanel`), строка «сейчас/дата+время» (`WhenRow`).

### Движок (engine)
- UI-независимый пакет: сервисы (`accounts`, `channels`, `posts`,
  `video`, `captions`), модель данных, шлюз Telegram.
- Работает в фоновом потоке со своим циклом событий asyncio
  ([ADR-0006](decisions/0006-desktop-pyside6.md)) — сетевые операции
  и кодирование видео не блокируют интерфейс.
- Отложенные посты публикует сервер Telegram
  ([ADR-0010](decisions/0010-posts-fire-and-forget.md)) — своего
  планировщика в приложении нет.

### Модуль обработки видео (`engine/video/`)
- Чистый модуль поверх `ffmpeg`: файл + параметры → обработанный файл
  и кадр-превью. Не знает про Telegram, каналы и БД.
  Подробно — [video-processing.md](../03-modules/video-processing.md).

### Шлюз Telegram (`engine/telegram/`)
- Два транспорта за одним фасадом: MTProto — публикация и чтение,
  Bot API — проверки и диагностика ([ADR-0011](decisions/0011-publish-via-userbot.md)).
  Подробно — [telegram-gateway.md](../03-modules/telegram-gateway.md).

### Хранилище данных
- SQLite в папке приложения, миграции Alembic при старте; секреты
  шифруются, ключ — в системном хранилище ОС
  ([ADR-0009](decisions/0009-config-secrets-in-db.md)).
  Модель — [data-model.md](../05-data/data-model.md).

## Таблица ответственности

| Компонент | Отвечает за | Не отвечает за | Зависит от |
|---|---|---|---|
| Интерфейс | формы, навигация, показ прогресса/ошибок | бизнес-логику, сеть | Движок (через EngineWorker) |
| Движок (сервисы) | бизнес-логика, валидации, оркестровка | отрисовку | БД, шлюз Telegram, модуль видео |
| Модуль видео | ffmpeg-конвейер, превью | Telegram, БД | ffmpeg/ffprobe |
| Шлюз Telegram | транспорты, перевод ошибок | бизнес-правила | aiogram, Telethon |
| Хранилище | схема, миграции, шифрование секретов | файлы медиа (только пути) | SQLite, keyring |
