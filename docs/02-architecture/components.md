# Компоненты системы

> Статус: 🟡 черновик-заглушка · Обновлено: 2026-06-30 · Владелец: olegosx

## Назначение документа

Перечислить компоненты приложения, их зону ответственности и связи между
ними. Уровень детализации — крупнее, чем модули, но конкретнее, чем общий
обзор.

## Состав (предварительно)

### Интерфейс (PySide6)
- [ ] Зона ответственности: нативное окно, формы, просмотр видео.
- [ ] Технология: PySide6 (Qt). Вызывает движок напрямую.

### Движок (engine)
- [ ] Зона ответственности: бизнес-логика, модель данных, оркестровка
      обработки видео, расписание, шлюз Телеграма. UI-независимый пакет.
- [ ] Связи: интерфейс (входящие вызовы), БД, фоновый поток.

### Фоновый поток движка
- [x] Зона ответственности: сетевые операции Telegram (проверки, отправка,
      отложки, вход userbot) — чтобы не блокировать интерфейс.
- [ ] Позже: обработка видео (`ffmpeg`), парсинг источников, генерация ИИ.
- Отложенные посты публикует сервер Telegram (ADR-0010) — своего
  планировщика в приложении нет.

### Модуль обработки видео
- [ ] Обёртка над `ffmpeg` (наработки переносим из `makeVideo`).

### Шлюз Telegram
- [ ] Обёртка над Bot API. Подробно — [telegram-gateway.md](../03-modules/telegram-gateway.md).

### Хранилище данных
- [ ] SQLite. Модель — [data-model.md](../05-data/data-model.md).
