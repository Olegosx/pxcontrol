# Поток данных

> Статус: 🟡 черновик-заглушка · Обновлено: 2026-06-30 · Владелец: olegosx

## Назначение документа

Показать путь контента от появления до публикации в канале — для каждого
из четырёх источников наполнения. Это помогает увидеть, где данные
создаются, хранятся, проверяются и отправляются.

## Потоки (заполнить деталями)

### Поток A. Ручной пост (ADR-0010: истина — канал, БД постов нет)
```
«Сейчас»:    Пользователь → Интерфейс → Движок → бот (Bot API) → Канал
Отложенный:  Пользователь → Интерфейс → Движок → userbot (schedule_date)
   → отложенная запись В КАНАЛЕ → публикует сервер Telegram
```

### Поток B. Генерация ИИ
```
Пользователь (тема, стиль) → Движок (фоновый поток) → Провайдер ИИ
   → черновик в БД → правка пользователем → расписание → публикация
```

### Поток C. Парсинг источника
```
Движок (по таймеру) → Источник (сайт/RSS/канал через MTProto) → нормализация
   → (опц. рерайт ИИ) → Очередь модерации (БД)
```

### Поток D. Модерация очереди
```
Пользователь → Интерфейс → Очередь модерации → одобрение
   → публикация как в потоке A («сейчас» или отложенно)
```
